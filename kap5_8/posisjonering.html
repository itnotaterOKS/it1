Timen i dag dekker et tema som ikke er prioritert i boka, men som er veldig nyttig: å bruke css til å posisjonere elementer på nettsiden.

Position

Hva?
Strukturerer posisjonen til et element på en nettside

Hvordan?
Bruker egenskapen position, som kan ha ulike verdier

Minner i tillegg om at alle elementer enten har display: inline (ved siden av, f.eks. < img > og < span >) eller block (under, f.eks < li > og < div >)

verdi beskrivelse
static default verdi, elementet følger rekkefølgen det legges i koden
absolute elementet ligger inne i et annet element, posisjonen justeres i forhold til dette
relative posisjonen sammenliknes med der det normalt ville vært plassert
fixed posisjonen bestemmes relativt til vinduet

Viser med div-bokser

static

absolute

relative

fixed

Her har de ulike:
.ramme{
   width: 500px;
   height: 300px;
   background-color: #F16699;
   margin: 20px auto;
   position: relative;
}
.ramme div{
   background-color: #EBF0F1;
   width: 80px;
   height: 80px;
  border: 1px solid black;
}
.stat{
   position: static;
}
.abs{
   position: absolute;
   left: 70px;
   top: 80px;
}
.rel{
   position: relative;
   top: 100px;
   left: 50px;
}
.fix{
   position: fixed;
   left: 0px;
   top: 150px;
}

Oppgaver:

Lag et .html-side med følgende elementer:
   < div >
     < h1 > Overskrift < /h1 >
     < img > Sett selv inn et bilde < /img >
     < table > Sett selv inn en tabell < /table >
     < ul > Sett selv inn en liste < /ul >
   < /div>

Bestem plasseringene til elementene med de ulike egenskapene vist over, legg merke til hva som skjer når du endrer css-koden. Gi gjerne elementene bakgrunnsfarge og/eller kantlinjer sånn at du ser hvor de starter og sluter.
Hva skjer hvis du øker bredden på innpakningsdiven?
Legg inn de samme verdiene for avstander (margin) og padding, men varier verdiene for position. Hva har position å si for resultatet?
